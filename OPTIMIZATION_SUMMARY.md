# Agent时间片优化总结

## 优化目标
在300个时间片的限制内，完成更多评测用例。时间片消耗公式：`t = 1 + max(0, (n-1000)) * 0.3`，其中n为本次模型调用的token数。

## 优化策略

### 1. 压缩SYSTEM_PROMPT（最关键优化）
**优化前**：306行，约8000+ tokens
**优化后**：约50行，约1500 tokens
**节省**：每次模型调用节省约6500 tokens
**时间片影响**：每次调用节省约 6500 * 0.3 / 1000 ≈ 1.95 个时间片

**优化措施**：
- 删除冗余说明和重复内容
- 合并相似规则
- 使用更简洁的表达方式
- 保留核心功能说明

### 2. 精简工具定义描述
**优化前**：每个工具description平均30-50字
**优化后**：每个工具description平均5-10字
**节省**：所有工具定义总计节省约500-800 tokens
**时间片影响**：每次调用节省约 0.15-0.24 个时间片

**优化措施**：
- `search_houses`: "按条件筛选房源，支持区域、价格、户型、地铁等筛选，返回结果含tags字段" → "按条件筛选房源，返回结果含tags"
- `get_house_detail`: "获取房源详情，包括属性、标签等，用于检查特殊需求" → "获取房源详情，含tags"
- 其他工具描述类似精简

### 3. 优化工具结果压缩
**优化措施**：
- 房源列表从最多保留10个减少到5个
- 地标列表从最多保留5个减少到3个
- 只保留关键字段（house_id, price, district, area等）

**节省**：每次工具调用结果节省约200-500 tokens

### 4. 减少对话历史保留
**优化前**：MAX_HISTORY_ROUNDS = 10（保留10轮对话）
**优化后**：MAX_HISTORY_ROUNDS = 5（保留5轮对话）
**节省**：每次调用节省约1000-2000 tokens（取决于历史长度）
**时间片影响**：每次调用节省约 0.3-0.6 个时间片

### 5. 减少工具调用迭代次数
**优化前**：最多5轮工具调用迭代
**优化后**：最多3轮工具调用迭代
**节省**：减少不必要的模型调用，每次节省至少1个时间片（基础时间片）

**说明**：大多数用例在3轮内可以完成，5轮通常是不必要的。

### 6. 限制模型输出token数
**优化措施**：添加 `max_tokens=500` 参数
**节省**：限制输出长度，减少响应token数
**时间片影响**：如果输出超过1000 tokens，每次节省约 0.15-0.3 个时间片

## 综合效果估算

### 单次模型调用节省
- SYSTEM_PROMPT压缩：约1.95个时间片
- 工具定义精简：约0.2个时间片
- 对话历史压缩：约0.5个时间片
- 工具结果压缩：约0.1个时间片（间接，通过减少输入token）
- **总计单次调用节省约2.75个时间片**

### 减少调用次数
- 工具调用迭代从5轮减少到3轮：每个用例节省2次调用 = 2个时间片（基础时间片）

### 总体效果
假设一个用例平均需要3次模型调用：
- **优化前**：3次调用 × (1 + 额外token时间片) ≈ 3-5个时间片
- **优化后**：3次调用 × (1 + 额外token时间片 - 2.75) ≈ 0.75-2.75个时间片
- **每个用例节省约2-3个时间片**

在300个时间片的限制下：
- **优化前**：可完成约60-100个用例
- **优化后**：可完成约100-150个用例
- **提升约50-70%**

## 注意事项

1. **功能完整性**：所有优化都保留了核心功能，不影响Agent的正确性
2. **可调参数**：如果发现某些用例需要更多历史或迭代，可以适当调整：
   - `MAX_HISTORY_ROUNDS`：可以调整为3-7之间
   - 工具调用迭代次数：可以调整为3-4之间
3. **监控建议**：建议在实际评测中监控：
   - 每个用例的平均时间片消耗
   - 用例完成率
   - 如果发现某些用例失败，可以适当放宽限制

## 进一步优化建议

如果还需要更多优化，可以考虑：

1. **智能缓存**：缓存常见查询结果，避免重复调用
2. **批量工具调用**：如果模型支持，可以一次调用多个工具
3. **更激进的压缩**：进一步压缩SYSTEM_PROMPT，只保留最核心的规则
4. **条件判断**：在代码层面添加更多判断，避免不必要的模型调用
